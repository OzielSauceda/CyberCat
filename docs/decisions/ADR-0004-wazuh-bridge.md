# ADR-0004 — Wazuh Bridge: Pull from Indexer via Asyncio Poller

**Date:** 2026-04-21  
**Status:** Accepted  
**Deciders:** Phase 8 planning session

---

## Context

Phase 8 adds the first real telemetry source: Wazuh. CyberCat needed to decide how to receive alerts from a running Wazuh stack. Wazuh exposes several mechanisms:

1. **Integrator → outbound webhook** — Wazuh calls a URL when a rule fires.
2. **Filebeat → direct ingest** — Filebeat ships raw log lines to a CyberCat HTTP endpoint, bypassing Wazuh's rule engine.
3. **Syslog forwarder** — Wazuh ships structured syslog to a CyberCat receiver.
4. **Indexer REST API (`_search`)** — CyberCat polls the OpenSearch-compatible indexer that already receives all Wazuh alerts via the manager's built-in Filebeat.
5. **Archives folder tail** — Read `/var/ossec/logs/archives/` directly from disk.

---

## Decision

Pull from the Wazuh Indexer REST API via a pull-mode asyncio poller running inside the FastAPI process. The poller uses `search_after` cursor pagination for restart safety.

---

## Rationale

| Criterion | Pull (chosen) | Push / webhook | Filebeat direct | Archives tail |
|---|---|---|---|---|
| Restart-safe | Yes — cursor in Postgres | No — alerts lost when backend is down | No — same gap | No — file offset not atomic |
| Asyncio-only (ADR-0002) | Yes | Requires inbound listener | Requires inbound listener | Requires inotify/poll task |
| Wazuh-down isolates CyberCat | Yes | No — manager must reach backend | No | No — file locked |
| Uses Wazuh rule engine | Yes | Yes | No — raw logs | Partial |
| No inbound port needed | Yes | No | No | Yes |

Pull wins on every operationally important dimension.

---

## Alternatives considered

### Integrator → webhook
Wazuh calls `POST /v1/ingest/wazuh` on each alert. **Rejected:** backend must be reachable from inside the compose network with its own auth/TLS; alerts queued by the manager are lost if the backend is down during an outage; tightly couples backend lifecycle to manager lifecycle.

### Filebeat → `/v1/events/raw`
Filebeat ships raw SYSLOG/JSON lines to CyberCat's existing ingest endpoint. **Rejected:** bypasses Wazuh's OOB rule engine entirely — CyberCat would have to re-implement all decoder/rule logic Wazuh already does. Defeats the integration story ("real detection engine operated by someone else").

### Syslog forwarder
Wazuh formats and ships syslog to a CyberCat UDP/TCP listener. **Rejected:** re-parsing text that Wazuh has already structured into JSON. Fragile to format changes. Requires a new persistent socket in the process.

### Archives folder tail
Read `/var/ossec/logs/archives/archives.json` via inotify or polling. **Rejected:** brittle file-format coupling; not API-contractual; not available when Wazuh runs in a separate container (volume mount required); offset file is a poor substitute for a proper cursor.

---

## Consequences

- Polling lag bounded by `WAZUH_POLL_INTERVAL_SECONDS` (default 5 s). Acceptable for IR — not for sub-second streaming.
- First-run must limit historical import via `WAZUH_FIRST_RUN_LOOKBACK_MINUTES` to avoid flooding the correlation engine.
- `alert._id` (OpenSearch document ID) is the dedup anchor. Guaranteed unique and stable within the indexer's shard assignment.
- `search_after = [iso_timestamp, _id]` is clock-skew-safe because `_id` lexically breaks ties.

---

## Security debt (tracked for Phase 9)

- **TLS verification off** (`WAZUH_INDEXER_VERIFY_TLS=false`). Wazuh self-signed certs are generated by the installer; pinning a CA bundle in v1 would require extracting and mounting the cert. Debt: flip to `true` + mount the CA in Phase 9.
- **`admin` indexer user**. Phase 9 provisions a `cybercat_reader` role via `internal_users.yml` + `roles_mapping.yml` with read-only access to `wazuh-alerts-*` only.

---

## Wazuh dashboard: deliberately not started

The Wazuh dashboard (`wazuh/wazuh-dashboard:4.9.2`) is a ~700 MB Kibana fork. It duplicates the analyst UI CyberCat is building with none of the custom correlation or response policy. Omitting it saves ~700 MB RAM (important under ADR-0003's 10 GB budget) and keeps the operator model focused on CyberCat as the primary interface. This is an explicit architectural decision, not an oversight.
